# 路线图

[English](./ROADMAP.md) | [简体中文](./ROADMAP.zh-CN.md)

这份路线图的目标，是让项目始终围绕“最小可行产品如何走向团队可用产品”来推进。

## 产品目标

打造一套帮助融资团队完成以下工作的系统：

- 记录投资人沟通内容
- 评估回答质量
- 沉淀融资知识资产
- 保持团队对外口径一致

## v0.3.0 内部试用版

目标：让产品能被团队稳定重复使用，而不只是单机演示。

计划内容：

- 支持异步音频处理和进度展示
- 提升真实转写场景下的角色重建能力
- 支持导出 Markdown 或 PDF 复盘结果
- 优化工作台里的错误提示、重试和失败恢复
- 增加更结构化的融资回答评分维度

成功标准：

- 30 到 60 分钟的会议上传后不会阻塞整条请求链路
- 转写角色重建足以应对内部试用数据
- 一个同事无需人工排障就能跑通一次完整复盘

## v0.4.0 团队协作版

目标：从本地 demo 工作流升级为多人共享使用。

计划内容：

- 升级为 PostgreSQL 加 `pgvector`
- 增加多用户工作区模型
- 增加标准答案审批流
- 支持主题趋势分析
- 加强不同大模型提供商之间的切换能力

成功标准：

- 能稳定搜索、比对多场会议
- 标准培训话术不需要直接改 JSON 文件
- 团队可以观察长期话术漂移和回答一致性变化

## v0.5.0 跨设备产品版

目标：支持真正的手机和电脑双端使用。

计划内容：

- Flutter 桌面和移动端客户端
- 会议音频对象存储
- 更完整的账号、权限和安全模型
- 托管部署与环境管理
- 复盘审批和管理者工作流

成功标准：

- 用户可以在桌面和手机上录音并查看复盘
- 录音和复盘结果不再依赖单台本地机器
- 产品具备进入托管试点的基础能力

## 当前暂不处理

- CRM 集成
- 投资人触达自动化
- 生产级权限细粒度控制
- 多语言转写纠错工作流
