# Changelog

这个文件用于记录项目的重要迭代。后续每次功能更新，建议继续按日期追加，方便团队回看演进过程。

## 2026-03-18

### 初始 MVP

- 搭建 `FastAPI + HTML/CSS/JavaScript` 的本地演示工作台。
- 提供会议录音复盘的核心 API，包括会议列表、问答复盘、主题库、标准话术和培训脚本。
- 支持手动粘贴转写文本后生成复盘结果。
- 增加示例对话、架构文档和数据流程文档，方便对内演示和继续立项。

### 模型增强与健康检查

- 接入 Moonshot / Kimi 的环境变量配置读取，避免把 API Key 写进代码。
- 新增 LLM 网关，在模型可用时生成更自然的会议总结、回答改进建议和统一培训话术。
- 扩展 `/api/health`，可以直接看到 `llm_enabled`、`llm_model`、`asr_enabled` 等运行状态。

### 音频输入与本地转写

- 接入 `faster-whisper`，支持本地音频转写。
- 支持上传 `mp3`、`wav`、`m4a`、`webm` 等音频文件进入分析链路。
- 前端支持浏览器内录音，录完后直接发起复盘。
- 让系统从“转写文本演示版”升级为“可直接吃音频的内部试用版”。

### 问答抽取与角色识别修复

- 优化无标签自然对话的问答抽取，不再强依赖 `投资人:` 和 `我:` 这样的显式标记。
- 增加自由文本 fallback，能够从自然转写中按问句特征重建 `问题 -> 回答`。
- 在 Kimi 可用时，先利用模型把自然转写重建为“投资人 / 我”的对话，再进入正式复盘流程。
- 保留 `raw_transcript_text`，方便回看原始转写和排查角色识别问题。

### 分享与交付

- 增加 `scripts/setup.ps1`、`scripts/run-demo.ps1` 和 `start-demo.bat`，方便同事快速启动。
- 增加 `COLLEAGUE_SETUP.md`，说明本地体验方式和 Kimi 配置方式。
- 生成可分享的演示包，便于内部转发和收集反馈。

### 测试与验证

- 增加音频上传、无标签转写、LLM 角色重建等测试覆盖。
- 当前已验证 `pytest` 通过，可用于继续迭代。
